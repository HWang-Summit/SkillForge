# 数据契约

## 跟踪表输入

支持 JSON 对象/数组、CSV 和 XLSX 首工作表。字段名不区分大小写；JSON 对象的键可作为缺失的 `record_id`。

| 字段 | 必需 | 含义 |
| --- | --- | --- |
| `record_id` | 是 | 稳定的源记录标识 |
| `title` | 是 | 文献标题 |
| `doi` | 否 | DOI，匹配时会规范化 |
| `pdf_path` | 是 | 相对 `--pdf-root` 或绝对 PDF 路径 |
| `pdf_status` | 否 | 若存在，仅值为 `downloaded` 的记录进入清单 |
| `zotero_item_key` | 否 | 已知 Zotero 父条目 key，优先级最高 |

## 文件产物

| 文件 | 说明 |
| --- | --- |
| `pdf_manifest.jsonl` | 准备阶段生成的不可变清单，含源文件哈希和父条目匹配结果 |
| `manual_review.jsonl` | 缺文件、匹配失败或匹配不唯一记录 |
| `import_results.jsonl` | Console 脚本按批追加的检查点，禁止手工编辑 |
| `batches.json` | 当前未完成记录的受限批次 |
| `console_batch-*.js` | 可粘贴到 Zotero Developer Console 的单批导入脚本 |
| `console_audit-*.js` | 可粘贴到 Zotero Developer Console 的单批只读核验脚本 |
| `attachment_audit.jsonl` | 导入后附件与父条目的全量核验结果 |
| `import_report.md`、`import_summary.json` | 中文统计报告与机器可读汇总 |

## 状态定义

- `pending_attach`：已唯一匹配且可导入。
- `manual_review`：需要人工决定，不能自动导入。
- `attached`：脚本成功创建 Zotero 托管附件。
- `skipped_same_hash`：检查点已证明相同父条目和源 SHA-256 已成功导入。
- `error`：该条导入失败，可在问题解决后重试。
- `review_existing_different_pdf`：同一父条目已有不同文件；不自动覆盖。
- `review_cross_parent_duplicate`：同一源哈希已关联到其他父条目；不自动复制。
- `review_existing_different_pdf`：父条目已有内容不同的 PDF；不自动新增第二份。
- `review_checkpoint_mismatch`：任务检查点与当前清单的父条目或源文件哈希不一致；停止自动处理。

## 恢复规则

不要删除 `import_results.jsonl`。再次运行 `plan` 时，`attached` 和 `skipped_same_hash` 记录会被排除；`error` 记录会被重新规划。若清单或源 PDF 改变，创建新的输出目录，避免混合不同任务。
